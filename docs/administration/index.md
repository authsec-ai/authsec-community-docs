# Administration

Manage your AuthSec platform with comprehensive administrative controls.

## Overview

The Administration section provides tools and guides for managing the core components of your AuthSec platform:

- **[Authentication](./authentication/)** - Configure SSO authentication methods (OIDC, OAuth, SAML)
- **[Users](./users/getting-started)** - User management, provisioning, and lifecycle
- **[RBAC](./rbac/getting-started)** - Role-Based Access Control and permissions
- **[Clients](./clients/)** - OAuth/OIDC client applications

## Key Features

### Authentication
Set up Single Sign-On (SSO) authentication methods for your users, including OIDC/OAuth 2.1 and SAML 2.0 providers.

### User Management
Manage user accounts, authentication methods, and user lifecycle. Control who has access to your applications and services.

### Role-Based Access Control (RBAC)
Define roles, permissions, and scopes to implement fine-grained access control across your organization.

### Client Management
Configure and manage OAuth 2.0/OIDC client applications that integrate with AuthSec.

## Getting Started

1. **[Configure Authentication](./authentication/)** - Set up SSO providers
2. **[Set up Users](./users/getting-started)** - Create and manage user accounts
3. **[Configure RBAC](./rbac/getting-started)** - Define roles and permissions
4. **[Register Clients](./clients/)** - Set up client applications
